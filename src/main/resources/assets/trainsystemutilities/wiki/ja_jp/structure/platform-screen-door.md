# ホームドア (Platform Screen Door)

```embed:item id=trainsystemutilities:platform_screen_door size=48 label=true
```

駅ホームに設置する可動式ホームドアです。4 方向の向きに対応し、列車の到着・発車に **自動で連動して開閉** します。1 台で 6 マス幅 (両端が柵、中央 4 マスが開く扉) です。

[[TOC]]

## 設置のしかた {#place}

1. インベントリでホームドアを手に持ちます。
2. ホームの縁に向けて **右クリック** すると設置されます。
3. ドアは **自分から見て左右に伸びる向き** で置かれます。設置したい向きに体を向けてから右クリックしてください。

> [!NOTE]
> ホームドアは 6 マス幅の 1 つのブロックです。**左右に十分な空きマス (左 3 / 右 2) が無いと設置できません** (アイテムは消費されません)。前後の壁ぎわではなく、開けたホーム縁に置いてください。

> [!TIP]
> 列車の扉位置に合わせて等間隔に並べると実物らしい見た目になります。[ホームフェンス](platform-fence.md)と併用してください。

## ドアの開け方 (重要) {#how-to-open}

**ホームドアを右クリックしても開きません。** 手動開閉は廃止されており、ドアは **列車が駅に到着/発車したときだけ自動で開閉** します。開くようにするには、下記の手順で駅 (鉄道管理ブロック) と連動させる必要があります。

## 駅と連動させる {#link}

ホームドアを列車と連動させる流れは次の 3 ステップです。フェンスとまったく同じ「メモリーカードのグループ」の仕組みを使います。

### ステップ 1: メモリーカードでドアをグループ登録 {#group}

1. **メモリーカード** を手に持ちます。
2. 連動させたいホームドア (連結しているフェンスも含めてよい) のどれか 1 つを **右クリック** します。
3. **隣り合って連結しているドア/フェンスが自動でまとめて登録** されます。チャットに「ホームドア群に ○個 追加 (計 ○個)」と表示されます。
4. カードを持っている間、登録済みメンバーが **緑の枠** でワールド上に表示されます。範囲を確認しましょう。
5. 余分なメンバーを外すときは、その部分を **Shift + 右クリック** します。
6. カードの登録数は、アイテムの **説明 (tooltip)** で確認できます。

### ステップ 2: カードを鉄道管理ブロックに入れる {#insert-card}

1. その駅の[鉄道管理ブロック](../railway-management.md)を **右クリック** して GUI を開きます。
2. 機能ドロップダウンから **ホームドア** の設定 popup を開きます。
3. popup 内の **カードスロット** に、ステップ 1 のメモリーカードを入れます (グループ登録済みのメモリーカードだけが入ります)。

### ステップ 3: 開閉の条件を設定する {#conditions}

ホームドア設定 popup で、列車のどのタイミングでドアがどう動くかを **条件** として登録します。条件は最大 16 件まで追加できます。

| 条件の項目 | 選べる値 | 意味 |
|---|---|---|
| 番線 | 番号 | 対象の番線 (MVP 版では番線判定は簡略) |
| イベント | 到着 (STOP) / 発車 (DEPART) | 列車がどうなった瞬間に発火するか |
| アクション | 開く / 閉じる / 帯色変更 | ドアに何をさせるか |

代表的な設定例:

- 「到着 → 開く」と「発車 → 閉じる」の 2 条件を登録 → 列車が着いたら開き、発車したら閉じる。

設定した動きは、popup 内の **テスト** 操作で列車を待たずに確認できます。帯色 (グループ全体のフェンス・ドアの帯の色) も同じ popup の色ボタンから変更できます。

> [!NOTE]
> 条件の登録先は「メモリーカードを入れた鉄道管理ブロック」です。列車検知は鉄道管理ブロックが行うため、**検知したい駅の鉄道管理ブロックにカードを入れてください**。
